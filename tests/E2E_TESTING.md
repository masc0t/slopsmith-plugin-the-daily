# E2E Testing Guide for The Daily Plugin

## Overview

The Daily is a 3D first-person dungeon rendered in a full-screen overlay
(`#ds-dungeon-overlay`, built by `screen.js`). The legacy 2D map / setlist /
complete / leaderboard / passport / shop views have been removed, so the
end-to-end tests now exercise the dungeon flow rather than DOM-snapshot views.

There are two layers of automated testing:

- **Backend (pytest / unittest)** — `tests/test_*.py` cover `routes.py`: the
  modifier engine, pool versioning, map generation, economy, shop, Wall of
  Fame, passport, and setlist-by-date. This is what CI runs.
- **Frontend smoke (Playwright)** — `tests/playwright/dungeon-smoke.spec.js`
  loads `screen.js`, enters the dungeon via `dsDungeonEnter`, and verifies the
  Quake-1 main menu renders, keyboard navigation works, the OPTIONS sliders
  adjust, and no console/page errors occur.

The dungeon must run from a **real HTTP origin** (Playwright's `page.setContent`
gives an opaque origin where `localStorage` throws), so the smoke spec spins up
a tiny `http.createServer` in `beforeAll` and serves the inlined test page. It
imports three.js from a CDN inside `dsDungeonEnter`; when that CDN is
unreachable (offline CI) the spec **skips** rather than failing.

For deeper, interactive dungeon debugging (walking, encounters, screenshots),
use the ad-hoc Playwright harnesses under `.scratch/dungeon-debug/` (gitignored)
— see the `reference-dungeon-headless-harness` note.

## Prerequisites

- Node.js (v16 or later) + npm
- Playwright Chromium: `npx playwright install chromium`
- Network access (for the three.js CDN import; otherwise the smoke spec skips)

## Running Tests

```bash
# Everything (backend coverage + e2e smoke)
npm test

# Backend only
npm run test:python

# Frontend smoke only
npm run test:e2e
# or a single file:
npx playwright test tests/playwright/dungeon-smoke.spec.js
```

Windows PowerShell runner: `./tests/run_e2e.ps1`

Useful Playwright flags: `--headed` (watch the browser), `--debug` (step
through), `npx playwright show-report` (last HTML report).

## Test Structure

```
tests/
├── playwright/
│   ├── dungeon-smoke.spec.js   # Quake main menu + OPTIONS smoke test
│   └── five_days_data.json     # Real-shaped /today fixture (5 days)
├── test-pages/
│   └── e2e_test.html           # Minimal shell that loads screen.js
├── run_e2e.sh / run_e2e.ps1    # Runners
├── test_*.py                   # Backend route tests (pytest/unittest)
└── E2E_TESTING.md              # This file
```

## Code Coverage

Backend coverage uses `coverage.py` and is generated by the standard suite:

```bash
npm run test:python                       # prints routes.py coverage summary
python -m coverage html --include=routes.py   # full HTML report → htmlcov/
```

## Troubleshooting

- **Smoke test skipped** — the three.js CDN was unreachable; the dungeon can't
  build without it. Run with network access to exercise the menu assertions.
- **Menu selectors not found** (`#ds-mm-items .ds-q-item`) — confirm
  `dsDungeonEnter` reached `_dsShowMainMenu` (WebGL must be available; headless
  Chromium provides SwiftShader by default).
- **localStorage SecurityError** — the page must be served over `http://`, not
  loaded via `setContent`/`file://`. The spec's built-in server handles this.
